# Plans

This folder contains implementation plans for features, refactors, and other work on christiandam.xyz.

## Conventions

- One markdown file per plan
- Name format: `YYYY-MM-DD-short-description.md`
- Plans should include: goal, approach, tasks, and status
- Mark plans as `completed` or `abandoned` when done

## Template

```markdown
# Plan: [Title]

**Date:** YYYY-MM-DD
**Status:** draft | in-progress | completed | abandoned

## Goal

What we're trying to achieve.

## Approach

How we'll do it.

## Tasks

- [ ] Task 1
- [ ] Task 2

## Notes

Any relevant context or decisions.
```
